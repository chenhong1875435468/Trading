# XAUUSD MT5 自动交易 EA 需求文档

## 1. 项目概述

本项目旨在构建一个用于黄金外汇（XAUUSD）交易的自动交易专家（Expert Advisor, EA），主要运行于 MetaTrader 5（MT5）平台。

项目的核心目标不是追求短期暴利，而是围绕“稳健、可控、可验证”的原则，设计一套以策略驱动为核心、风控优先、界面友好的自动交易系统。

> 注意：本项目仅用于策略研究、自动化交易系统开发与回测验证，不构成任何投资建议。实盘交易存在亏损风险。

## 2. 核心目标

- 构建一个适用于 XAUUSD 的 MT5 自动交易 EA。
- 以稳定盈利为长期目标，不追求高风险暴利。
- 所有订单必须带止损，禁止无止损裸单。
- 不采用抗单、无限加仓、马丁格尔等高风险模式。
- 控制好盈亏比、胜率、回撤和单笔风险。
- 支持策略参数化，便于回测、优化和复盘。
- 配备美观、清晰、用户友好的图形化界面。

## 3. 初始交易条件

- 交易品种：XAUUSD
- 运行平台：MT5
- 初始资金：100 USD
- 交易风格：稳健型、低回撤、风险可控
- 策略方向：以明确入场信号和严格风控为基础

## 4. 策略设计原则

策略应优先满足以下原则：

- 信号清晰：入场、出场、止损、止盈逻辑必须明确。
- 风控先行：任何交易动作都必须先计算风险，再判断是否开仓。
- 单单止损：每一笔订单必须设置止损。
- 不抗单：价格触发止损后严格退出，不通过补仓摊低成本。
- 可回测：策略逻辑必须能够在 MT5 中进行历史回测。
- 可解释：每次开仓应能说明触发了哪些信号。
- 可配置：关键参数应允许用户在 EA 输入项或界面中调整。

## 5. 候选策略方向

后续可以围绕以下方向逐步筛选和组合策略：

- 趋势跟随：例如均线方向、ADX 趋势强度、价格突破。
- 动量确认：例如 RSI、MACD、KDJ、CCI 等指标共振。
- 波动率过滤：例如 ATR 判断止损距离、过滤低波动行情。
- 关键位置交易：例如前高前低、支撑阻力、整数关口。
- 多周期确认：例如大周期判断方向，小周期寻找入场点。
- 时间过滤：避开流动性差、点差异常或重大新闻前后的时段。

第一阶段不急于堆叠大量指标，应优先找到一个逻辑简单、回测稳定、风险可控的基础策略，再逐步增强。

### 5.1 第一版策略建议：行情状态识别 + 双模式执行

第一版策略建议采用“先识别行情状态，再选择交易模式”的框架：

- 偏趋势行情：采用趋势跟随。
- 偏震荡行情：采用区间反转。
- 状态不清晰：不交易。

这样做的原因是，XAUUSD 经常在趋势推进和高波动震荡之间切换。单一策略容易在不适合的行情中连续亏损，而行情状态过滤可以减少低质量交易。

#### 5.1.1 行情状态识别

建议使用多个条件共同判断，不依赖单一指标。

趋势行情识别条件：

- ADX 高于趋势阈值，例如 25。
- EMA 快线在 EMA 慢线之上且斜率向上，判断多头趋势。
- EMA 快线在 EMA 慢线之下且斜率向下，判断空头趋势。
- 价格运行在布林带中轨同一侧，且布林带宽度扩张。
- ATR 高于近期均值，说明波动率足够支撑趋势交易。

震荡行情识别条件：

- ADX 低于震荡阈值，例如 20。
- EMA 快慢线缠绕或斜率较小。
- 价格在布林带上下轨之间来回运行。
- ATR 未明显扩张。
- 最近一段行情存在清晰的高低边界。

不交易条件：

- ADX 位于 20 到 25 之间，趋势强度不明确。
- 刚发生大阳线或大阴线，但没有形成延续结构。
- 点差过大、波动异常或临近重大消息。
- 多个指标互相矛盾。

#### 5.1.2 趋势跟随模式

趋势跟随模式适合在方向清晰、波动扩张时使用。

多单思路：

- 大周期方向偏多。
- 快速 EMA 位于慢速 EMA 上方。
- ADX 高于趋势阈值。
- 价格回踩 EMA 或布林带中轨后重新向上。
- RSI 不处于极端超买，避免追在短期高点。

空单思路：

- 大周期方向偏空。
- 快速 EMA 位于慢速 EMA 下方。
- ADX 高于趋势阈值。
- 价格反抽 EMA 或布林带中轨后重新向下。
- RSI 不处于极端超卖，避免追在短期低点。

止损建议：

- 使用 ATR 止损，或放在最近结构高低点外侧。
- 趋势模式的目标盈亏比不应太低，建议至少 1.5:1。

适用特点：

- 胜率可能不如区间反转高。
- 但单笔盈利空间更大，更依赖盈亏比。
- 适合捕捉黄金单边快速行情。

#### 5.1.3 区间反转模式

区间反转模式适合在趋势不强、价格围绕区间上下沿波动时使用。

多单思路：

- ADX 低于震荡阈值。
- 价格接近布林带下轨或区间下沿。
- RSI 进入超卖区域后开始回升。
- K 线出现企稳信号，例如下影线、吞没或收回关键价位。
- 止盈目标优先看区间中轨或区间上沿。

空单思路：

- ADX 低于震荡阈值。
- 价格接近布林带上轨或区间上沿。
- RSI 进入超买区域后开始回落。
- K 线出现受阻信号，例如上影线、吞没或跌回关键价位。
- 止盈目标优先看区间中轨或区间下沿。

止损建议：

- 放在区间边界外侧，并叠加 ATR 缓冲。
- 如果价格突破区间并开始扩张，应立即止损，不抗单。

适用特点：

- 通常更容易做出较高胜率。
- 单笔盈利空间较小，需要严格控制止损。
- 最大风险是把趋势启动误判成震荡反转，因此必须使用趋势过滤。

#### 5.1.4 第一版默认策略倾向

基于“100 USD 小资金、稳健、不抗单”的项目目标，第一版建议采用以下优先级：

1. 先判断是否适合交易，不适合就空仓。
2. 如果趋势明确，只做顺趋势回调单，不追涨杀跌。
3. 如果趋势不明确但区间清晰，只做区间边界反转。
4. 如果既不是清晰趋势，也不是清晰区间，则不交易。

第一版策略不建议频繁交易，宁可少交易，也要提高信号质量。

## 6. 风控要求

风控是本项目的最高优先级。

基础风控规则：

- 每笔交易必须设置止损。
- 单笔风险建议控制在账户权益的 0.5% 到 2% 之间。
- 当账户资金较小（例如 100 USD）时，默认应使用极低风险参数。
- 每日最大亏损达到阈值后，当日停止交易。
- 连续亏损达到阈值后，暂停交易并等待人工确认或下一个交易周期。
- 点差过大时禁止开仓。
- 滑点超过设定阈值时应拒绝或谨慎执行交易。

需要重点关注的指标：

- 最大回撤
- 胜率
- 盈亏比
- 期望收益
- 连续亏损次数
- 单日最大亏损
- 订单平均持仓时间

## 7. EA 功能需求

### 7.1 交易功能

- 自动识别交易信号。
- 自动开仓、平仓。
- 自动设置止损和止盈。
- 支持固定手数和按风险比例计算手数。
- 支持只做多、只做空、双向交易模式。
- 支持交易时间段限制。
- 支持最大持仓数量限制。
- 支持每日交易次数限制。
- 支持点差过滤。

### 7.2 参数配置

建议至少支持以下参数：

- 交易手数模式：固定手数 / 风险比例
- 固定手数大小
- 单笔风险比例
- 止损计算方式：固定点数 / ATR / 结构位
- 止盈计算方式：固定盈亏比 / 固定点数 / 信号反转
- 最小盈亏比
- 最大点差
- 最大滑点
- 交易时段
- 每日最大亏损
- 连续亏损暂停阈值
- 魔术号（Magic Number）

### 7.3 日志与复盘

- 记录每次信号触发原因。
- 记录每笔订单的开仓、平仓、止损、止盈信息。
- 记录因风控条件未开仓的原因。
- 支持在 MT5 日志中输出关键决策信息。
- 后续可扩展为导出 CSV 或本地报告。

## 8. 界面需求

EA 应配备美观且用户友好的界面，便于观察当前交易状态和策略状态。

界面应展示：

- 当前账户余额、净值、浮动盈亏。
- 当前交易品种、点差、交易时段状态。
- 当前策略方向判断。
- 当前是否允许开仓。
- 今日交易次数、今日盈亏。
- 当前持仓信息。
- 最近一次信号触发原因。
- 风控状态，例如是否触发暂停交易。

界面操作建议：

- 一键启用 / 暂停自动交易。
- 一键平仓当前 EA 管理的订单。
- 快速切换保守 / 标准 / 激进参数模式。
- 清晰区分正常状态、警告状态和禁止交易状态。

## 9. 技术实现方向

- 开发语言：MQL5
- 运行平台：MetaTrader 5
- 版本管理：Git
- 首要目标：先实现可回测的策略内核，再完善界面和辅助功能。
- 初版源码文件：`XAUUSD_Regime_EA.mq5`
- 初版实现约束：所有 EA 逻辑暂时放在一个 `.mq5` 文件中，不拆分多文件模块。

建议模块划分：

- `Strategy`：策略信号判断。
- `RiskManager`：风险控制、仓位计算、交易限制。
- `TradeManager`：订单执行、修改、平仓。
- `Panel`：图形化界面展示和按钮操作。
- `Logger`：日志记录和调试输出。
- `Config`：参数配置和默认值管理。

当前初版虽然采用单文件实现，但代码内部仍应按策略、风控、交易执行、面板、工具函数等区域组织，便于后续维护。

### 9.1 初版 EA 功能范围

`XAUUSD_Regime_EA.mq5` 初版包含：

- 行情状态识别：趋势上涨、趋势下跌、震荡、不明确。
- 趋势跟随模式：趋势明确时，只做顺势回调后的延续单。
- 区间反转模式：震荡明确时，只在布林带或区间边缘寻找反转单。
- 不确定行情过滤：ADX 中性区、指标矛盾、区间不清晰时不交易。
- 固定手数和按风险比例计算手数两种模式。
- 每笔订单自动设置止损和止盈。
- 点差过滤、交易时段过滤、最大持仓数限制。
- 每日最大亏损限制、每日交易次数限制、连续亏损暂停。
- 新 K 线触发交易判断，减少同一根 K 线重复开仓。
- 图表面板展示账户、策略、风控和最近动作状态。
- 面板按钮支持启停自动交易、平仓、切换保守 / 标准 / 激进模式。

初版建议先在 MT5 策略测试器中回测，不建议直接上实盘。

## 10. Git 托管约定

本项目需要通过 Git 进行版本管理。

建议约定：

- `master` 或 `main`：稳定主分支。
- `develop`：日常开发分支。
- `feature/*`：具体功能开发分支。
- `fix/*`：问题修复分支。

提交信息建议使用清晰的中文或英文描述，例如：

- `docs: 初始化需求文档`
- `feat: 添加基础交易信号模块`
- `feat: 添加风险控制模块`
- `fix: 修复止损计算问题`
- `refactor: 优化订单管理逻辑`
